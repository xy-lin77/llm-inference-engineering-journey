# 量化（FP32/BF16 → INT8/INT4）
## 1. 目标
通过线性映射将高精度浮点数（FP32/BF16）压缩到低比特整数域（INT8/INT4），以可控精度损失，换取显存降低、推理加速、部署成本下降。

| 类型 | 位宽 | 单参字节 | 核心收益 |
|------|------|----------|--------------------------|
| FP32 | 32bit | 4B       | 精度损失                 |
| BF16 | 16bit | 2B       | 平衡精度与显存           |
| INT8 | 8bit  | 1B       | 显存减半，GPU专项优化加速 |
| INT4 | 4bit  | 0.5B     | 显存骤降，需精度优化      |

---

## 2. 量化原理

### 1. 在哪里量化

- 要量化：词嵌入层（仅量化存储，使用时反量化回浮点）、注意力模块中的 `Q, K, V, O` 投影层、FFN 中的两个线性层。
- 可量化：QK^T相似度计算、alpha * V
- 不量化：LN、Softmax、GeLU / SiLU

### 2. 核心原理

- **基本思想**：通过尺度因子 `scale` 和零点 `zero_point`，建立浮点数与定点整数之间的双向映射。
- **量化 (Q)**：`x_quant = round(x_float / scale) + zero_point`
- **反量化 (D)**：`x_float = (x_quant - zero_point) * scale`
- 量化 `Q` 和反量化 `D` 是一对互逆操作，使得 `D(Q(x_float)) ≈ x_float`。

### 3. 对称量化与非对称量化

- **线性层权重 (W)**：其分布通常近似关于零点对称，适合采用**对称量化**（`zero_point` 通常为0），计算更高效。
- **激活值 (X)**：通常来自上一层的 GeLU 等非线性函数，其分布不均匀且偏向正值，理论上**非对称量化**更精准，但**对称量化**因性能优势也常被采用。
- **独立量化**：权重 `W` 和激活值 `X` 的数值分布和范围差异巨大，因此必须使用各自独立的 `scale` 和 `zero_point` 进行量化。

2.  量化核心逻辑（无矛盾，贴合实操）
    - 线性层计算：int8×int8（纯低精度，省显存/提速度）
    - 激活层衔接：线性层int8输出无需单独反量化，直接进入激活层（暂用int8，不单独反量化）
    - 反量化原则：N个线性层+激活层全程不反量化，仅在所有计算完成后，做1次反量化（int8→bf16）

3.  精度敏感环节要求
    - 敏感环节（QK相似度、softmax、alpha×V、LN等）：需在进入前反量化为bf16，用bf16×bf16高精度计算，避免误差放大
    - 非敏感环节（线性层、激活层传递）：保持int8低精度，无需中途反量化

4.  关键澄清
    - 无int8×bf16实际运算：激活层是int8先参与计算，最终统一反量化为bf16，而非每个激活层单独反量化
    - 一次反量化覆盖全程：抵消所有线性层+激活层的累积误差，无需多次反量化（避免回到全精度，失去量化意义）
    - Q(AB)的近似性：严格数学上Q(AB)=(QA)B（线性映射性质）；实操中因B（权重）可量化（QB≈B），近似Q(AB)≈(QA)(QB)（实现低精度计算）


- **计算重排与混合精度**：
    - 将浮点权重矩阵 `W` 分解为低精度（如 INT8）的 `Q(W)` 和一个或多个浮点尺度因子 `D`（通常是逐通道的向量），使得 `W ≈ D ⊗ Q(W)`。
    - 利用乘法结合律，原计算 `Y = W @ X` 可以重排为 `Y = D ⊗ (Q(W) @ X)`。
    - 这使得计算可以分为两步：
        1.  **矩阵乘法**：执行高效的混合精度计算 `Q(W) @ X`，即 W8A16（INT8 权重 @ BF16 激活值），得到一个中间结果。
        2.  **反量化**：通过廉价的逐元素/逐通道乘法，将中间结果乘以尺度因子 `D`，恢复其数值范围，得到最终的 BF16 输出。

- **保护敏感环节**：在进入对数值精度高度敏感的计算环节前，必须确保数据流是高精度的（如 BF16）。这些环节包括：
    - **统计计算**：如 Layer Normalization 中的均值和方差计算。
    - **非线性激活**：如 GeLU、SiLU 等激活函数。
    - **概率计算**：如计算注意力权重时的 Softmax 函数。
    - **其他精密操作**：如 RoPE、注意力得分与 V 的乘积等。
### 2. 两种量化方式
- 对称量化（权重量化首选）：基于权重最大绝对值计算scale，无zero_point，适配正负对称分布。
- 非对称量化（激活值常用）：基于数值全范围计算scale和zero_point，适配非对称分布。
### 3. INT4 关键差异
- 取值范围[-8,7]，离散值少、误差大，需分组量化（per-block）节省参数。
### 4. 量化粒度（影响精度）
- per-tensor：整层共享scale/zero_point，简单但误差大。
- per-channel：每通道独立量化，卷积层常用，精度提升。
- per-group/per-token：LLM INT4标配，平衡精度与效率。
### 5. 直接低精度计算的支撑
- 数学层面：线性映射可交换，先量化再计算与先计算再量化结果一致（误差仅来自舍入）。
- 硬件层面：专用计算单元（如TensorCore）原生支持INT8/INT4直接计算，无需反量化。

## 3. 量化误差来源
1. 舍入误差：浮点连续值→整数离散值的固有偏差。
2. 截断/饱和误差：浮点极值超出整数范围，离群值是精度暴跌主因。
3. 累积误差：多层误差叠加，LLM更明显。
4. 校准偏差：校准数据与真实场景分布不一致。

## 4. 精度恢复方法（核心方案）
| 方案 | 核心逻辑 | 优势 | 适用场景 |
|------|----------|------|----------|
| PTQ（训练后量化） | 预训练模型直接量化，少量校准数据 | 无训练、速度快 | 快速部署 |
| QAT（量化感知训练） | 训练中模拟量化噪声，微调适应低精度 | 精度几乎无损 | 高精度任务 |
| QLoRA | 4-bit量化 + LoRA微调，冻结基础模型 | 显存↓95%，精度接近FP16 | 大模型单卡微调 |
| QAD（量化感知蒸馏） | FP16教师模型指导INT4学生模型 | 比PTQ准、比QAT快 | 多阶段微调模型 |
| 离群值处理 | 单独量化/保留敏感离群值通道 | 针对性解决精度暴跌 | 大模型量化 |

## 5. 实战流程
1. 模型分析（权重分布、离群值、敏感层）→ 2. 选择方案 → 3. 校准（100-500条场景数据）→ 4. 量化 → 5. 评测 → 6. 精度修复。

---

## 6. 关键误区
1. 量化≠压缩：不仅减显存，更改变计算格式、触发硬件加速。
2. INT4不一定比INT8差：GPTQ/AWQ优化后，INT4精度接近INT8，显存再省50%。
3. 量化可加速：依赖内存带宽、硬件算力、缓存命中率三重提升。
